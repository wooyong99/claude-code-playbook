# CLAUDE.md 프로젝트 컨텍스트 설정

## 목적

CLAUDE.md 템플릿의 프로젝트 전용 플레이스홀더를 실제 프로젝트 정보로 채운다.

채워야 할 플레이스홀더:

| 플레이스홀더 | 내용 |
|-------------|------|
| `{프로젝트명}` | 프로젝트 이름 |
| `{프로젝트 한 줄 설명}` | 프로젝트를 한 문장으로 |
| `{비즈니스 목표 및 핵심 기능 설명}` | 비즈니스 목표 목록 |
| `{사용자 유형 · 인터페이스 · 진입점}` | 사용자·인터페이스·진입점 테이블 |
| `{디렉토리 구조 및 각 디렉토리 역할}` | 디렉토리 구조와 각 역할 |

---

## Step 1. CLAUDE.md 현재 상태 파악

작업 전 CLAUDE.md를 읽어 어떤 플레이스홀더가 아직 비어 있는지 확인한다.
이미 채워진 항목은 건너뛴다.

---

## Step 2. 인터뷰

비어 있는 플레이스홀더에 대해서만 질문한다. **한 번에 모든 질문을 던지지 않고**, 항목별로 하나씩 묻되 사용자가 여러 정보를 한꺼번에 제공했다면 해당 항목들을 묶어서 처리한다.

### 질문 순서

1. **프로젝트명 · 한 줄 설명**
   > "프로젝트 이름과 한 줄 설명을 알려주세요."

2. **비즈니스 목표**
   > "핵심 기능이나 비즈니스 목표를 bullet point로 나열해 주세요."

3. **사용자 · 인터페이스 · 진입점**
   > "사용자 유형과 각 인터페이스, 진입점(모듈명, URL 등)을 알려주세요."
   >
   > 예시: "샵 방문 고객 / Storefront / :app:storefront"

4. **디렉토리 구조**
   > "주요 디렉토리 구조를 알려주세요. 생략하면 프로젝트를 직접 탐색해 추론합니다."
   >
   > 사용자가 생략하면 `find` · `ls`로 최상위 디렉토리를 탐색하고, 각 디렉토리의 역할을 추론하여 사용자에게 확인 후 사용한다.

---

## Step 3. CLAUDE.md 작성

수집한 정보를 아래 포맷에 맞게 변환하여 CLAUDE.md의 플레이스홀더를 교체한다.

### 비즈니스 목표 포맷

```
- {목표 1}
- {목표 2}
...

### 사용자와 인터페이스

| 사용자 | 인터페이스 | 진입점 |
|--------|-----------|--------|
| {사용자 유형} | {인터페이스명} | {진입점} |
```

### 프로젝트 구조 포맷

````
```
{디렉토리}/    # {역할 설명}
{디렉토리}/    # {역할 설명}
...
```
````

---

## Step 4. 완료 확인

CLAUDE.md 수정 후, 변경된 섹션만 사용자에게 보여주고 내용이 맞는지 확인을 요청한다.
수정이 필요하면 해당 항목만 다시 받아 교체한다.
